## 🛡️ Introduction: 鉄の斧と、電子の魔導書

シュタークは、途方に暮れていた。

村の重要な水源を汚す「魔族（課題）」が現れた。物理的な斧では太刀打ちできない。
「AIは魔法のランプだ」と教わったが、画面に「助けてくれ」と打ち込んでも、返ってくるのは冷たい環境規制のガイドラインだけ。

「伝え方（プロンプト）」がわからない。

力が足りないんじゃない。自分の想いを、AIという巨大な機関が理解できる「呪文」に変換する方法を知らないだけなのだ。

昔は、魔法の書を覚えればなんとかなると思っていた。
今は、パソコンを眺めれば何かできると思っていた。
……どちらも、動かし方がわからなければ、ただのガラクタだ。

そんな彼の前に、一冊の古びた、しかし新しい輝きを放つ魔導書が差し出された。

**『Neu Kodex（ノイ・コデックス）』**

これは、あなたの「混沌（モヤモヤ）」を、世界を変える「一般攻撃魔法（MVPプロンプト）」へと精錬する物語である。

---

## 🔮 この魔導書の「術式」（機能）

### Phase 0：書架の探索
モヤモヤを入力すると、QuestQueriesシリーズ全18作のREADMEを参照し、
Claude APIが「近いアプリ」を提案する。既存の魔法で解決できるなら、それが最速だ。

### Phase 1〜3：三章の巡礼（錬成フロー）
既存アプリでは解決できない場合、ゼロから錬成フローへ進む。

1. **認知の初期化（ショーモン）：** 最初の一言で「知・情・意」の最適ルートへ導く。
2. **三章の巡礼：**
   - **📘 青の章：魔法使い（知）** — 現状を整理し、解決すべき「問い（What）」を見出す。
   - **📕 赤の章：戦士（意）** — ターゲット（Who）を射抜き、実行の武器を研ぐ。
   - **📗 緑の章：僧侶（情）** — 継続の根（Core Value）を植え、安心を届ける。
3. **叡智の結晶化：** 汎用AIで即座に使える **MVPプロンプト** を生成して渡す。

## 🚀 使い方

1. `index.html` をブラウザで開く（GitHub Pages推奨）。
2. モヤモヤをそのまま入力する。
3. 書架に近いアプリがあれば試してみる。なければ錬成フローへ。
4. 三章をクリアしてMVPプロンプトを受け取る。
5. ChatGPT・Gemini・Claudeに貼り付けて使う。

## 📝 Architecture

このアプリはY Combinator式ピッチ分析（Who / What / Core Value）を
ヒーローズ・ジャーニーとして体験させる魔導書です。

### 章の順番が変わる仕組み

オープニングの一問によって3章の順番が動的に変わります。
ユーザーは気づかないまま、最も答えやすい順番で進んでいく設計です。

| オープニング選択 | 章の順番 |
|---|---|
| 自分が困っている | 緑 → 青 → 赤 |
| 誰かが困っている | 赤 → 青 → 緑 |
| 世界をよくしたい | 青 → 赤 → 緑 |

### 書架（ライブラリ機能）

起動時にqq-collectiveの全リポジトリのREADMEを一括fetchし、
モヤモヤに近いアプリをClaude APIでマッチングする。

```
ユーザーがモヤモヤを入力
  ↓
18個のREADMEをfetch（raw.githubusercontent.com）
  ↓
Claude APIが近いアプリを提案
  ↓ 見つかった → リンク表示
  ↓ 見つからない → 錬成フローへ
```
